## 一、MarkAsRead 所有权校验修复

### 问题

`MarkAsRead(id, userID)` 使用 `WHERE id = ? AND user_id = ?` 查询但未检查 `RowsAffected`。当通知不存在或不属于当前用户时，GORM 的 `Update` 不报错（0 rows affected），导致任意用户可标记他人通知为已读，HTTP 返回 200。

### 修改文件

**1. `backend/services/notification-service/repository/notification.go`**

- 添加 `"fmt"` 导入
- `MarkAsRead` 方法增加 `RowsAffected` 检查，0 行时返回 `"通知不存在或无权操作"`

**2. `backend/services/notification-service/handler/notification.go`**

- `MarkAsRead` handler 区分错误类型：
  - `"通知不存在或无权操作"` → HTTP 404
  - 其他错误 → HTTP 500

### 测试验证

- `TestMarkAsRead` — 正确用户标记已读 ✓
- `TestMarkAsRead_WrongUser` — 错误用户返回 404，通知状态不变 ✓

## 二、comment.created 事件发布者

### 问题

`comment.created` 事件已在 notification-service 注册消费者，但 interaction-service 创建评论后未发布该事件。

### 修改文件

**3. `backend/services/interaction-service/service/content.go`**

- `CommentService` 增加 `db *gorm.DB` 和 `publisher *events.Publisher` 字段
- `Create()` 成功后异步发布 `comment.created` 事件：
  - 查 `articles` 表获取 `author_id` 和 `title`
  - 查 `users` 表获取评论者 `username`
  - 使用 goroutine 异步发布，不阻塞 HTTP 响应
  - 发布失败仅记录日志

## 关键设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 获取 article author/title | `*gorm.DB` 直接查 `articles` 表 | 最小改动，避免新建 repository |
| 获取用户名 | `*gorm.DB` 直接查 `users` 表 | 数据权威，不需要 handler 传参 |
| 事件发布时机 | goroutine 异步 | 不阻塞 HTTP 响应，匹配 content-service 模式 |
| 发布失败处理 | 仅 `log.Printf`，不中断主流程 | fire-and-forget 通知模式 |
| 点赞事件过滤 | 仅 `targetType=="article"` | 通知类型已限定为 article.liked |
| Nil publisher 保护 | 所有发布点 `if s.publisher != nil` | 支持测试和 RabbitMQ 不可用时的优雅降级 |

## 六、网关路由补充 & notification-service 纯消息队列化

### 问题

网关存在以下缺口：
- `permission`(8004)、`search`(8005) 在注册表中但服务不存在（幽灵条目）
- `notification-service` 应在后台消费消息队列写入通知，不作为 HTTP 服务暴露
- `content-service` 缺少 `category`、`publish` 路由
- `interaction-service` 缺少 8 条路由（likes、collections、delete comment），且 `/api/comments/:id/like` 路由指向不存在的端点

### 修改文件

**9. `backend/services/notification-service/main.go`**

重构为纯消息队列消费者：
- 连接 MySQL，执行 `Notification` 表迁移
- 创建 Consumer + Repository + Service
- 调用 `svc.StartListening()` 注册 4 类事件监听（article.published、comment.created、user.followed、article.liked）
- 无 HTTP 服务器，`select {}` 阻塞等待消息

**10. `backend/api-gateway/routes/router.go`**

全面修正网关路由：

| 变更类型 | 详情 |
|----------|------|
| 删除幽灵服务 | `permission`(8004)、`search`(8005) 从注册表移除 |
| 补充 content-service | `GET /api/articles/category/:category`（公开）+ `POST /api/articles/:id/publish`（认证） |
| 补充 interaction-service | `DELETE /api/comments/:id`、`POST /api/likes`、`DELETE /api/likes`、`GET /api/likes/status`、`POST /api/collections`、`DELETE /api/collections/:article_id`、`GET /api/collections/status`、`GET /api/collections` |
| 修复路由错误 | `POST /api/comments/:id/like` → `POST /api/likes` |
| 修复拼写错误 | `GET /api/users/ :id/following` → `GET /api/users/:id/followings` |

**11. `backend/services/notification-service/go.mod`**

- `go mod tidy` 添加 `gorm.io/driver/mysql` 依赖

**12. `backend/services/content-service/go.mod`**

- `go mod tidy` 补全 `go.sum`
